# 수익링크 심층 분석 — 제휴 링크 자동 수익화

> 분석 대상: `https://bit.ly/4fA6ZLZ`
> 최종 착지: Mytheresa 코리아, The Row Astra 레더 숄더백 (₩4,180,000)

---

## 1. 이 링크의 정체 — URL 해부

bit.ly는 겉포장일 뿐이고, 리다이렉트를 끝까지 따라가면 아래 URL로 착지합니다.

```
https://www.mytheresa.com/kr/ko/women/the-row-astra-leather-shoulder-bag-black-p00999131
  ?dplink=true
  &pub_name=2975314        ← 퍼블리셔(제휴사) 이름 ID
  &pub_id=4441350          ← 퍼블리셔 계정 ID
  &chn=aff1 &tarea=int     ← 채널=제휴, 지역=국제
  &src=cj                  ← 제휴 네트워크 = CJ Affiliate
  &cmp=aff_cj_int
  &utm_medium=affiliate_cj_int
  &cjevent=58a3a493...     ← ★ CJ 클릭 추적 ID (수익 귀속의 핵심)
  &slink_id=38541618-...   ← ★ 스마트링크(자동 제휴) 레코드 ID
  &ptyp=dlg                ← 딥링크(특정 상품으로 바로 연결)
```

핵심은 두 개입니다.

- **`cjevent`** — 누군가 이 링크를 클릭한 순간 CJ가 발급하는 고유 클릭 도장입니다. 이 사람이 30일 안에 Mytheresa에서 구매하면, 그 매출이 `pub_id=4441350` 퍼블리셔에게 귀속되고 **커미션이 지급**됩니다.
- **`slink_id`** — "smart link", 즉 **아무 상품 URL이나 자동으로 제휴 링크로 바꿔주는 서비스**를 거쳤다는 흔적입니다. 사람이 일일이 링크를 만든 게 아니라, 링크 하나 던지면 시스템이 CJ 추적을 붙여준 것입니다.

**즉 이 링크는 "명품 상품 링크를 제휴 추적 링크로 자동 변환 → bit.ly로 단축 → 배포"** 한 것이고, 클릭·구매마다 퍼블리셔가 수수료를 버는 구조입니다. 정확히 슬릭딜·딜 인플루언서들이 쓰는 방식입니다.

## 2. 돈이 도는 원리

| 요소 | 내용 |
|---|---|
| 네트워크 | **CJ Affiliate** (Commission Junction) |
| 커미션율 | **판매액의 8%** (Mytheresa 기준) |
| 쿠키 기간 | **30일** (클릭 후 30일 내 구매까지 인정) |
| 최소 지급액 | 없음 |
| 예시 | 저 가방(₩418만) 1개 판매 시 커미션 **약 ₩334,000** |

명품 해외직구는 **객단가가 높아** 전환 1건의 커미션이 큽니다. 전환율이 낮아도(보통 1~3%) 단가가 커서 수지가 맞습니다. 효종님 시스템이 이미 **프리미엄 브랜드 위주로 큐레이션**하도록 만들어 둔 것과 정확히 맞물립니다.

## 3. 이런 링크를 만드는 두 가지 방법

### 방법 A — 머천트별 딥링크 (직접 제휴)
CJ·Rakuten·Impact에서 **특정 머천트(Mytheresa, 노드스트롬 등)에 개별 승인**을 받고, 그 네트워크의 딥링크 생성기로 상품 URL을 추적 링크로 바꿉니다.
- 장점: **커미션이 가장 높다**(중간 수수료 없음), 정확한 추적.
- 단점: 머천트마다 승인 필요, 링크 형식이 네트워크마다 다름.

### 방법 B — 스마트링크 애그리게이터 (자동)
**Skimlinks / Sovrn Commerce** 같은 서비스에 한 번 가입하면, **4만8천+ 머천트 · 50여 개 네트워크**를 자동 제휴해 줍니다. 어떤 상품 URL이든 던지면 알아서 추적을 붙여 줍니다(이번 링크의 `slink_id`가 바로 이 방식).
- 장점: **머천트별 승인 불필요**, 링크 하나로 전 사이트 커버 → 즉시 대규모 적용.
- 단점: 서비스가 커미션의 일부(약 25%)를 가져감. 순수익은 방법 A보다 낮음.

**권장: 방법 B로 즉시 전체 수익화를 켜고(승인 대기 없이), 매출 큰 핵심 머천트(Mytheresa·메이시스 등)만 방법 A로 직접 승인받아 커미션을 극대화.** 두 방식은 병행 가능합니다.

## 4. 딜온도에 적용 — 이미 구현했습니다

효종님 시스템은 지금 딜 링크를 뿌리지만 **전부 맨 링크라 수익이 0원**이었습니다. 방금 **제휴 링크 래퍼(`scraper/affiliate.py`)** 를 붙였습니다.

동작:
- 딜보드·텔레그램에 나가는 모든 구매 링크를 **자동으로 제휴 추적 링크로 변환**합니다.
- **중복제거용 원본 URL은 그대로 보존**하고, 표시·발송용 `buy_url`만 새로 만듭니다(시스템 안정성 유지).
- 우선순위: 머천트 딥링크 템플릿(방법 A) → 스마트링크(방법 B) → 원본. **설정이 없으면 원본이 나가므로 지금도 정상 동작**합니다.
- 딜보드에 **제휴 고지 문구**를 추가했습니다(법적 필수).

즉 코드 변경 없이 **설정값만 넣으면 그 순간부터 전 딜이 수익 링크**로 나갑니다.

## 5. 활성화 절차 (효종님이 할 일)

**즉시 켜기 (방법 B, 며칠 내)**
1. **Skimlinks** 또는 **Sovrn Commerce** 가입 (콘텐츠/딜 퍼블리셔로).
2. 발급받은 퍼블리셔 ID를 GitHub 시크릿에 추가:
   - `RADAR_SKIMLINKS_ID` = 내 Skimlinks ID   (또는 `RADAR_SOVRN_KEY`)
3. 끝. 다음 실행부터 전 딜 링크에 자동으로 커미션 추적이 붙습니다.

**수익 극대화 (방법 A, 핵심 머천트)**
1. **CJ Affiliate** 가입 → Mytheresa·노드스트롬 등에 개별 제휴 신청.
2. **Rakuten** → 메이시스 / **Impact** → The Outnet (2부의 상품피드 제휴와 동일 계정 재사용).
3. 승인되면 각 머천트의 딥링크 형식을 `data/affiliate.csv`에 한 줄씩 등록(파일 안에 CJ/Rakuten/Impact 템플릿 예시 있음).
4. 그 머천트 딜은 스마트링크 대신 **직접 딥링크**로 나가 커미션이 최대가 됩니다.

## 6. 기존 수익화 3안과의 관계

앞서 정리한 수익화는 (1) 유료 멤버십, (2) 배대지 제휴였는데, 이 **제휴 링크가 사실상 세 번째이자 가장 즉시·수동적인 기둥**입니다.

| 방식 | 착수 난이도 | 수익 시점 | 성격 |
|---|---|---|---|
| **제휴 링크** | 낮음(설정만) | 즉시 | 클릭·구매당 커미션, 완전 자동 |
| 유료 멤버십 | 중 | 사용자 확보 후 | 구독 반복 매출 |
| 배대지 제휴 | 중 | 물량 확보 후 | 배송 리퍼럴 |

**제휴 링크를 먼저 켜는 게 맞습니다** — 지금 뿌리는 모든 딜이 곧바로 수익원이 되고, 사용자 수가 적어도 명품 전환 1건이 크기 때문입니다.

## 7. 유의사항 (정직하게)

- **고지 의무**: 제휴 링크임을 반드시 밝혀야 합니다(공정위·FTC). 딜보드에 문구를 넣었고, 텔레그램 채널 소개에도 한 줄 넣는 걸 권장합니다.
- **전환율은 낮습니다**: 클릭 대비 구매는 보통 1~3%. 트래픽(구독자)이 커질수록 수익이 붙습니다. 초기엔 소액입니다.
- **쿠키 종속**: 사용자가 최종적으로 다른 제휴 링크를 거치거나 앱으로 이탈하면 귀속이 깨질 수 있습니다.
- **스마트링크 수수료**: 방법 B는 편한 대신 커미션 일부를 떼입니다. 매출 큰 머천트는 방법 A로 옮기세요.
- **네트워크 승인 조건**: 일부 네트워크는 '운영 중인 사이트/채널'을 요구합니다 — 이미 GitHub Pages 딜보드가 있으니 심사에 유리합니다.
