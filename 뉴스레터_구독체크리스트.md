# 뉴스레터 구독 체크리스트

> **이게 프로모션 코드 경쟁력의 전부다.**
>
> 결제 코드는 세일 페이지에 안 뜬다(실측 확인). 리테일러가 구독자 메일로 먼저 뿌리고,
> 그걸 받은 사람이 슬릭딜에 올린다. **메일함에 도착한 시점이 슬릭딜보다 앞선다.**
>
> 코드 추출기는 완성됐다. 이제 **구독한 만큼만 잡힌다.**
> 등록된 발신자는 58곳인데, 구독을 안 하면 0곳이나 마찬가지다.

---

## 준비

이메일 수집용으로 만든 **전용 Gmail 계정**으로 구독한다.
개인 메일에 섞으면 감당이 안 된다.

구독 방법은 대부분 같다. 사이트 맨 아래(footer)로 내려가면
`Sign up for emails` / `Email signup` / `Newsletter` 입력칸이 있다.
이메일 주소만 넣으면 끝이다.

> 팁: 가입 시 "여성/남성/키즈" 관심사를 물으면 **전부 체크**한다.
> 카테고리별로 다른 코드를 뿌리는 경우가 많다.

## 현실적인 목표

| 구독 수 | 기대 |
|---|---|
| 5곳 | 코드가 가끔 잡힌다. 체감 어려움 |
| **15곳** | **주 몇 건. 여기부터 의미 있다** |
| 30곳+ | 거의 매일. 슬릭딜보다 앞서는 게 실감난다 |

**한 번에 다 하려 하지 말 것.** 1순위 9곳부터 하고, 며칠 돌려보며
실제로 코드가 잡히는지 확인한 뒤 늘리면 된다.

## 확인 방법

구독 후 하루 이틀 지나면 메일이 오기 시작한다.

```
9_이메일수집.bat
```

이렇게 나오면 성공이다.

```
[email] macys.com                2건
[email] theoutnet.com            1건
[email] 총 3건 추출

  [FLASH] 85점 84% [코드 EXTRA20] Further Reductions: Up to 80% Off
```

**"총 0건"이 나와도 정상이다.** 아직 메일이 안 왔거나, 온 메일에 코드가 없는 것뿐이다.
할인 정보가 없는 배송 안내·신상품 메일은 자동으로 걸러진다.

## 등록 안 된 곳을 구독했다면

`scraper/sources_email.py` 의 `SENDER_MAP` 목록에 도메인을 추가하면 된다.
서브도메인(`email.xxx.com`)은 자동으로 매칭되므로 루트 도메인만 적으면 된다.

```python
SENDER_MAP = {d: d for d in [
    "macys.com",
    "새로운사이트.com",     # ← 여기 추가
    ...
]}
```

여기 없는 발신자의 메일은 **무시된다.** 개인 메일이 섞여도 안전하다는 뜻이기도 하다.

## 주의

- **스팸함 확인.** 쇼핑 메일은 스팸으로 분류되기 쉽다.
  Gmail에서 스팸함에 들어간 메일을 "스팸 아님"으로 몇 번 표시해주면 개선된다.
- **수신 거부 링크는 누르지 말 것.** 코드가 안 온다.
- 구독을 많이 하면 메일이 하루 수십 통 온다. 전용 계정을 쓰는 이유다.
