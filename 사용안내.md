## 자동 실행 등록

`10_자동실행등록.bat` 을 **마우스 오른쪽 클릭 → 관리자 권한으로 실행**.

윈도우 작업 스케줄러에 4개 작업이 등록된다.

| 작업 | 주기 | 대상 |
|---|---|---|
| HotdealRadar_T1 | 15분 | Amazon, Woot, eBay |
| HotdealRadar_T2 | 30분 | Nordstrom Rack, Zappos 등 |
| HotdealRadar_T3 | 2시간 | 중간 우선순위 25곳 |
| HotdealRadar_T4 | 6시간 | 나머지 89곳 |

- 창이 뜨지 않고 백그라운드로 돈다
- PC를 껐다 켜도 자동 재개된다
- **PC가 켜져 있어야 돈다** (슬립 모드에서는 멈춘다)
- 해제하려면 `11_자동실행해제.bat`

24시간 끊김 없이 돌리려면 클라우드 서버가 필요하다(월 5천원 수준).
미국 새벽 딜까지 잡으려면 결국 그쪽이지만, 우선 이대로 며칠 써보고 판단해도 된다.

## 며칠 돌려본 뒤 다듬을 것

실전 데이터가 쌓여야 판단할 수 있는 것들이다. **지금 미리 손대지 말고**,
알림을 받아보면서 아래 증상이 나타나면 그때 조정하면 된다.

### 알림이 너무 많다

`scraper/filter_engine.py` 에서 기준선을 올린다.

```python
def should_publish(deal) -> bool:
    return deal.passed_hard_filter and deal.score >= 50   # 55 또는 60으로
```

### 알림이 너무 적다 / 좋은 딜을 놓친다

같은 값을 45 정도로 내린다.
또는 자주 사는 브랜드가 `기타`로 분류되고 있는지 확인한다.
`CATEGORY_KEYWORDS` 에 브랜드명을 추가하면 점수가 오른다.

### 엉뚱한 딜이 온다

어떤 사이트에서 왔는지 본다. 특정 사이트가 계속 이상하면
`data/watchlist.csv` 에서 그 줄의 `crawl_allowed` 를 `NO` 로 바꾸면 제외된다.

### 알림 시간대가 안 맞는다

`scraper/scheduler.py` 의 발행 슬롯을 생활 패턴에 맞게 고친다.

```python
PUBLISH_SLOTS = [(8, 10), (13, 16), (20, 23)]
```

FLASH(85점 이상)는 이 슬롯과 무관하게 즉시 발송된다.

## 90일 뒤 자동으로 좋아지는 것

지금은 가격 이력이 없어서 스펙의 **가격 히스토리 점수(25점)** 가 꺼져 있다.
대신 나머지 75점을 100점으로 환산해 쓰고 있다.

매 스캔마다 가격이 쌓이고 있으므로, 상품별로 **관측 3회 이상 + 기간 3일 이상**이
모이면 그 상품부터 자동으로 정상 채점으로 전환된다.
"역대 최저가 갱신" 같은 판정이 그때부터 가능해진다.

진행도는 실행 로그에 나온다.

```
알림 이력 45건 · 가격 관측 518건 / 0일 축적
```
